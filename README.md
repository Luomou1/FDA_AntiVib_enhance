# FDA_AntiVib_enhance

`FDA_Antivib` 是一个面向白光扫描干涉数据分析的桌面应用，当前支持 `FDA` 与 `PhaseGap` 两条处理流程，并集成了有效扫描范围自动检测能力。项目从原有 `Phase_Gap` 工作基础上拆分出来，目标是把抗振场景下的干涉分析流程收敛为一个可直接运行、可视化、便于后续维护的独立仓库。

## 功能概览

- 提供基于 `PySide6` 的桌面 GUI，便于交互式选择数据与参数
- 支持 `FDA` 与 `PhaseGap` 分析模式切换
- 顶部提供独立的“平面分析”和“台阶分析”一级模块
- 平面分析读取单个二维高度 `.txt`，支持平面/鲁棒平面/二次曲面校准及完整形貌绘图
- 台阶分析读取独立的二维高度 `.txt`，支持三点调平、自动高低层分割、双区域高度差和去噪/不去噪切换
- 自动检测 active scanning range，并支持在检测结果基础上分别设置左、右扩展帧数
- 支持自动估计全局 `K0`，也支持手动指定固定 `K0`
- 支持自适应 Hann/Hamming 分析窗：逐像素估计干涉包络中心，并按左右不同宽度构造非对称窗
- 提供频谱长度档位选择，兼顾分析精度与运算成本
- 输出高度、相位、拟合斜率等关键结果，并配套图形查看

## 处理流程

1. 读取图像序列或体数据
2. 沿扫描维度计算干涉调制强度，自动确认 active scanning range
3. 自动估计全局 `K0`，或读取用户指定的固定 `K0`
4. 对逐像素信号去直流
5. 施加窗函数并执行频谱分析；自适应窗会先用 Hilbert 包络定位每条曲线的有效中心
6. 在全局 `K0` 附近截取局部拟合窗口
7. 对局部相位解包裹并拟合得到 `g0` 与 `phi0`
8. 计算基础高度 `h = -g0 / 2`
9. `FDA` 模式下执行整数级次修正，得到 `h_prime`
10. `PhaseGap` 模式下在 FDA baseline 结果上继续做 phase-gap connect / fit / finalization，输出最终 `h_prime`

## 主要输出

- `h`：由频域相位斜率得到的基础高度图
- `h_prime`：最终修正后的高度图
- `phi0`：全局 `K0` 处的相位图
- `g0`：局部相位拟合斜率
- `active_range`：自动识别出的有效扫描帧范围

## 表面高度分析

顶部导航顺序为：

`工作台 | 平面分析 | 台阶分析 | 设置`

“平面分析”和“台阶分析”分别选择自己的高度文本文件，数据与分析状态互不共享。文本文件应为规则的二维数值矩阵，默认按 `1 nm/单位` 换算。

- 平面分析输出原始三维形貌、校正并去噪后的三维形貌、二维高度图和行/列轮廓。
- 台阶分析先在图中选择三个同层非共线参考点，再选择“去噪”或“不去噪”进行分析；分层完成后依次框选两个矩形区域，程序按“区域一均值 - 区域二均值”计算台阶高度。

## 频谱长度策略

界面支持以下频谱长度档位：

- `1x`：FFT 长度等于有效帧数
- `256 / 512 / 1024`：固定目标 FFT 长度，默认 `512`；若有效帧数大于目标长度，程序会使用有效帧数以避免截断信号
- `2倍 / 4倍 / 8倍补0`：直接按有效帧数倍增后补零

如果启用“扩大有效范围”，程序会在自动检测到的范围左、右两侧分别按设定帧数扩展，并自动裁剪到数据边界内。扩展后的范围会同时参与自动 `K0` 估计和正式分析。

自适应窗模式下，窗函数最大权重对准每条干涉曲线的包络峰。左右边界由包络弱阈值分别确定，因此可处理左右拖尾不同的不对称干涉信号；窗外样本不会从原始数据中删除，只是在频谱输入中被余弦窗降权到零，避免背景或其他包络段污染当前主峰分析。

## 项目结构

```text
FDA_Antivib/
├─ main.py                  # 桌面应用入口
├─ pyproject.toml           # 项目元数据与依赖声明
└─ app/
   ├─ core/                 # FDA / PhaseGap 核心分析逻辑
   ├─ gui/                  # 主界面、线程与交互窗口
   ├─ pipeline/             # 数据读取、session、active range 检测
   └─ plotting/             # 绘图与可视化辅助
```

## 环境要求

- Python `3.11+`
- 推荐在 Windows 环境下运行桌面界面

## 安装方式

先创建虚拟环境，再安装项目依赖：

```powershell
python -m venv .venv
.venv\Scripts\Activate.ps1
pip install -U pip
pip install -e .
```

如需安装测试依赖：

```powershell
pip install -e .[test]
```

## 运行方式

在项目根目录执行：

```powershell
python main.py
```

## Windows 打包与在线更新

程序名称为“数据分析”，图标资源位于 `assets/app_icon.ico`。本地打包命令：

```powershell
powershell -ExecutionPolicy Bypass -File scripts\build_windows_exe.ps1
```

构建产物：

- `dist\数据分析.exe`：PyInstaller 单文件 GUI 程序
- `dist\installer\数据分析-0.1.0-setup.exe`：安装向导；仅在本机已安装 Inno Setup 且 `ISCC.exe` 可用时生成

主界面右上角提供“检查更新”按钮。在线更新基于 GitHub Releases：

1. 在 `Luomou1/FDA_AntiVib_enhance` 发布形如 `v0.1.1` 的 Release
2. 上传 `数据分析-0.1.1-setup.exe` 或其他 `.exe` / `.msi` / `.zip` 安装包资产
3. 客户端发现远程版本高于当前版本后才下载；已下载且大小一致的安装包会直接复用

在线更新请求不内置 GitHub Token。若仓库保持私有，普通客户端无法读取 Release API；需要将 Release 所在仓库设为公开，或改为公开的更新清单/下载源。

程序启动时会清理 `%TEMP%` 和本应用更新缓存中的本应用安装包，并删除未完成的 `.part` 下载文件；不会扫描或删除其他软件的安装包。构建脚本成功生成安装器后，会自动清理 `dist\installer` 下的旧 `.exe`，只保留最新安装器。

如果需要上传 Release 安装包，使用：

```powershell
powershell -ExecutionPolicy Bypass -File scripts\upload_release_installer.ps1 -Tag v0.1.0
```

该脚本会把最新安装器复制到系统临时目录，以 `DataAnalysis-版本号-setup.exe` 的英文文件名上传，上传完成后删除临时副本，不在项目目录里遗留额外安装包。
